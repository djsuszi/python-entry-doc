Python jako pierwszy język

  • prosty w nauce (naturalne konstrukcje)
  • czytelny (wygodna recenzja, czytanie logiki)
  • interpretowany – szybkie i łatwe programowanie oraz testowanie
  • sprawne debugowanie
  • brak konieczności zarządzania pamięcią (automatyczne)
  • kod jest przenośny (zadziała tam, gdzie jest interpreter)
  • szybki dewelopment (nie ma kompilacji)
  • bogata biblioteka standardowa, dużo modułów trzecich

Główne cechy

  • Obiektowy: wszystko jest obiektem
  • Imperatywny: wykonuje instrukcję wiersz po wierszu
  • Interpretowany: posiada maszynę wirtualną, która uruchamia kod z plików
  • Język wysokiego poziomu: skupiamy się na logice biznesowej
  • Wcięcia są częścią składni – zamiast klamer, średników (jak w C czy Javie)
  • Kod znajduje się w plikach tekstowych, które nie wymagają kompilacji
  • Posiada interaktywną konsolę: umożliwia testowanie kodu na żywo

Konsola interaktywna

Konsola interaktywna pozwala na bezpośrednie wydawanie poleceń języka bez konieczności pisania pliku programu. Umożliwia inspekcję obiektów, testowanie i uruchamianie kodu "na żywo".

Aby uruchomić wbudowaną konsolę interaktywną, należy w powłoce systemowej (w Windows wiersz poleceń - cmd, w Linux/MacOS emulator terminala) wydać polecnie: python 

piotrek@dell:~
    $ python
    Python 2.7.15rc1 (default, Nov 12 2018, 14:31:15) 
    [GCC 7.3.0] on linux2
    Type "help", "copyright", "credits" or "license" for more information.
    >>> print('hello world!')
    hello world!

Standardowa konsola może być niewygodna w użyciu - zalecaną alternatywą jest ipython. Posiada ona kolorowanie składni, autouzupełnianie oraz dodatkowe funkcje niedostępne w wersji standardowej. 

piotrek@dell:~
    $ ipython3
    Python 3.6.7 (default, Oct 22 2018, 11:32:17) 
    Type 'copyright', 'credits' or 'license' for more information
    IPython 7.2.0 -- An enhanced Interactive Python. Type '?' for help.
    
    In [1]: print("hello world!")                                                                                     
    hello world!

ipython można zainstalować przy użyciu manadżera pakietów pip wydając polecenie: pip install ipython

W przypadku systemu Windows przdatna będzie również powłoka GitBash posiadająca podobne funkcje i wygląd jak powłoki Linuxowe.

Uruchamianie programów

Kod programów umieszczony jest w tekstowych plikach z rozszerzeniem py i do ich uruchomienia wymagany jest interpreter, który będzie w stanie wykonać instrukcje zawarte w tym pliku.

Najprostszym sposobem na uruchomienie programu z pliku jest wydanie polecenia python plik.py 

piotrek@dell:~/projects/isa
    $ python3 hello.py 
    hello world

Python - podstawy

Obiekty

Struktury danych w Pythonie reprezentowane są przez obiekty. Obiekt to obszar pamięci, w którym przechowywana jest integralna jednostka danych. Obiekty posiadają:

  • id - identyfikator
  • typ - czyli rodzaj obiektu (rodzina)
  • wartość - jakie dane reprezentują

Obiektami w Pythonie są np.liczby, ciągi znakowe, kolekcje (sekwencje) ale także funkcje, klasy, moduły. Obiekty można przekazywać jako argumenty funkcji, wykorzystywać ich atrybuty lub funkcje (zwane metodami)
Podstawową inspekcję obiektów można wykonać funkcjami wbudowanymi:

  • id() - zwraca unikalny identyfikator obiektu
  • type() - zwraca typ obiektu
  • print() - wyświetla reprezentację tekstową
  • repr() - wyświetla reprezentację dla dewelopera
  • dir() - wyświetla listę atrybutów obiektu
  • help() - wyświetla opis pomocy (dokumentację), tzw. docstring

Zmienne

Zmienne w Pythonie należy rozumieć jako etykiety, które jedynie przechowują lokalizację (adres obiektu) w pamięci. Na jeden obiekt może wskazywać wiele zmiennych. Operacje na zmiennych wykonywane są na obiektach.
Do przypisania obiektu do zmiennej używa się operatora = 

In [1]: a = 3                                                                                                     

In [2]: b = 'to jes napis'                                                                                        

In [3]: a = b                                                                                                     

In [4]: a                                                                                                         
Out[4]: 'to jes napis'

In [5]: b                                                                                                         
Out[5]: 'to jes napis'

Typy danych

Proste

Typy danych można w Pythonie podzielic na proste (atomowe) jak np. typ liczby całkowita czy typ funkcji oraz na kolekcje. Kolekcje to obiekty, które mogą przechowywać inne obiekty.

Obiekty mogą być zmienne lub niezmienne. Cechą szczególną jest wykonywalność (funkcje, klasy).

int

int - typ liczby całkowitej, obiekt niezmienny.
Konstruktorem jest funkcja int() 

In [11]: a = 1                                                                                                    

In [12]: b = int('3')                                                                                             

In [13]: a + b                                                                                                    
Out[13]: 4

float

float - typ liczby rzeczywistej (zmiennoprzecinkowej), obiekt niezmienny.
Konstruktorem jest funkcja float() 

In [14]: f = 5.4                                                                                                  

In [15]: x = float('6.6')                                                                                         

In [16]: f * x                                                                                                    
Out[16]: 35.64

bool

bool - typ reprezentujący wartość logiczną prawda/fałsz
Konstruktorem jest funkcja bool()
Argumentem funkcji może być dowolny obiekt.
Wartości liczbowe odpowiadające liczbie 0 oraz puste kolekcje traktowane są jako Fałsz (wartość False), pozostałe obiekty zazwyczaj jako Prawda (wartość True). 

In [17]: bool('to jest napis')                                                                                    
Out[17]: True

In [18]: bool(0)                                                                                                  
Out[18]: False

In [19]: True == False                                                                                            
Out[19]: False

NoneType

None - reprezentuje wartość nieustaloną

Kolekcje

Kolekcje można nazwać kontenerami - mogą zawierać inne obiekty.
Cechą karakterystyczną kolekcji jest możliwość sprawdzenia wielkości zbioru funkcją len()
Pobieranie elementów z kolekcji będących sekwencjami odbywa się za pomocą ideksów (ideksem jest liczba naturalna)
sekwencja[n] - pobiera n-ty element ze zbioru

str

Typ napisowy str reprezentuje ciągi znaków alfabetów używanych przez ludzi. Obiekty napisów są niezmienne. Konstruktorem napisów jest funkcja str()
Tekst można formatować (wstawiać wartości) przy uzyciu metody format() 

In [27]: n = 'to jest napis'                                                                                      

In [28]: n = """ to "też" 'jest' napis"""                                                                         

In [29]: n = ( 
    ...:     'a to jest ' 
    ...:     'napis ' 
    ...:     'wielowierszowy' 
    ...: )                                                                                                        

In [30]: n                                                                                                        
Out[30]: 'a to jest napis wielowierszowy'
In [1]: '{0} to bardzo {jaki} język programowania'.format('Python', jaki='fajny')                                 
Out[1]: 'Python to bardzo fajny język programowania'

list

Listy do kontener zmienny, który może przechowywać dowolną liczbę obiektów różnych typów. Lista zachowuje kolejność elementów i mogą się one powtarzać. Elementy z listy można dodawać i usuwać.
Konstruktorem listy jest funkcja list() 

In [31]: l1 = ['a', False, None, 6.6, [1, 2, 3]]                                                                  

In [32]: l2 = list('to jest napis')                                                                               

In [33]: l2                                                                                                       
Out[33]: ['t', 'o', ' ', 'j', 'e', 's', 't', ' ', 'n', 'a', 'p', 'i', 's']

tuple

Tupla (krotka) to kontener bardzo podobny do listy, ale jest obiektem niezmiennym. Raz utworzony nie pozwala na dodawania lub usuwanie elementów zbioru. Same obiekty mogą jednak się zmieniać.
Konstruktorem tupli jest funkcja tuple() 

In [36]: tuple('napis')                                                                                           
Out[36]: ('n', 'a', 'p', 'i', 's')

In [37]: krotka = (1, None, False, 'abc')                                                                         

In [38]: len(krotka)                                                                                              
Out[38]: 4

Sekwencje obsługują operacje wycinanania podzbiorów: 

In [51]: napis = 'ABCDEFGHIJK'                                                                                    

In [52]: napis[0] # pierwszy element                                                                              
Out[52]: 'A'

In [53]: napis[-1] # ostatni element                                                                              
Out[53]: 'K'

In [54]: napis[3:5] # podzbiór elementów od trzeciego do piątego                                                  
Out[54]: 'DE'

In [55]: napis[3:] # podzbiór elementów od trzeciego do końca zbioru                                              
Out[55]: 'DEFGHIJK'

In [56]: napis[0:12:2] # co drugi element ze zbioru od pierwszego do dwunastego                                   
Out[56]: 'ACEGIK'

In [57]: napis[::-1] # pełny zbiór w porządku odwrotnym                                                           
Out[57]: 'KJIHGFEDCBA'

dict

Słowniki mają charakter mapy - są nieuporządkowaną kolekcją zestawu elementów klucz: wartość
Klucze w słowniku nie mogą się powtarzać. Kolejność elementów słownika może zmienić się w trakcie pracy programu.
Konstruktorem jest funkcja dict()
Słowniki nie są sekwencjami, pobieranie elementów odbywa się przez podanie obiektu klucza zamiast indeksu:
slownik[klucz] - zwraca element przypisany pod obiekt klucz lub rzuca wyjątek KeyError 

In [42]: s = {'klucz': 'wartość', None: 1, 5: ['a', 'b', 'c']}                                                    

In [43]: dict([('a', 'b'), (1, 3)])                                                                               
Out[43]: {'a': 'b', 1: 3}

In [44]: s['klucz']                                                                                               
Out[44]: 'wartość'

Operatory

Operatory są częścią składni i służą do wkonywania operacji na obiektach.
Można je podzielić na kilka rodzajów:

  • Przypisania (wynik wyrażenia z prawej przypisany jest do zmiennej z lewej):
  • =
  • Arytmetyczne (można łączyć z operatorem przypisania):
  • +, -, *, /, //, %, **
  • Porównania i przynależności:
  • ==, !=, <, >, <=, >=, is, in
  • Logiczne:
  • and, or, not

Kontrola przebiegu programu

Instrukcja warunkowa

Instrukcja warunkowa if pozwala na podejmowanie decyzji w programie.

if wyrażenie:
blok instrukcji
elif:
inny blok instrukcji
else:
alternatywny blok instrukcji

Bloki elif oraz else są opcjonalne.
wyrażenie może być dowolnie skomplikowane - może być np. wywołaniem funkcji 

In [58]: if len('abc') >= 3: 
    ...:     print('Ciąg ma 3 znaki lub więcej') 
    ...: else: 
    ...:     print('Ciąg ma mniej niż 3 znaki') 
    ...:                                                                                                          
Ciąg ma 3 znaki lub więcej

Pętla for

Pętla for używana jest do iteracji po kolekcjach.

for element in kolekcja:
instrukcja A
else:
instrukcja B

Zmienna element przyjmuje kolejne obiekty z kolekcji przy każdym obrocie pętli.
Blok else zostanie wywołany nawet przy pustej kolekcji (gdy nie wykona się ani razu instrukcja A), chyba, że pętla zostanie przerwana instrukcją break 

In [63]: for el in []: 
    ...:     print(el) 
    ...: else: 
    ...:     print('pusta kolekcja') 
    ...:                                                                                                          
pusta kolekcja

In [64]: for el in 'ABC': 
    ...:     if el == 'C': 
    ...:         break 
    ...:     print(el) 
    ...: else: 
    ...:     print('pusta kolekcja') 
    ...:                                                                                                          
A
B

Pętla while

Pętli while używa się gdy nie wiemy, ile razy instrukcje zostaną wykonane.

while wyrażenie:
instrukcja A
else:
instrukcja B

Pętla while może nie wykonać się ani razu, jeśli inicjalnie warunek nie będzie spełniony.
Opcjonalny blok else wykonywany jest po zakończeniu obrotów pętli, chyba, że zostanie ona przerwana instrukcją break
Instrukcja continue przerywa wykonywanie bieżącej pętli i przechodzi do następnej iteracji. 

In [68]: while len(a): 
    ...:     print(a.pop()) 
    ...: else: 
    ...:     print('Kolekcja jest już pusta') 
    ...:                                                                                                          
3
2
1
Kolekcja jest już pusta

Funkcje

Funkcje są obiektami wykonywalnymi, w ciele których można zawrzeć blok instrukcji. Zostaną one wykonane po wywołaniu funkcji, co następuje po wpisaniu znaków () po jej nazwie.

Obowiązującą konwencją nazewnictwa funkcji jest lower_case_with_underscore

Deklaracja funkcji

Python posiada szereg funkcji wbudowanych w bibliotece standardowej, ale można tworzyć własne.
Instrukcją deklaracji funkcji jest def

def nazwa_funkcji():
ciało funkcji

Funkcje mogą zwracać dowolne obiekty. Użycie instrukcji return zwraca obiekt i kończy działanie funkcji.
Główną zaletą i powodem stosowania funkcji jest zmniejszenie ilości i powtarzalnosci kodu. 

In [69]: def funkcja(): 
    ...:     print('to jest ciało funkcji') 
    ...:     return True 
    ...:                                                                                                          

In [70]: funkcja()                                                                                                
to jest ciało funkcji
Out[70]: True

Obsługa parametrów

Funkcje mogą przyjmować parametry. Są dwa rodzaje parametrów:

  • pozycyjne (wymagane)
  • nazwane (opcjonalne, z wartoscią domyślną)

Przekazując argumenty do wykonywanej funkcji należy najpierw podać wszystkie pozycyjne, a dopiero potem ewentualnie (wybrane) argumenty nazwane.
W przypadku parametrów nazwanych, dla których nie przekaże się argumentu, nastąpi użycie wartości domyślnych.

Wartościami domyślnymi nie powinny być obiekty zmienne (np. listy). 

In [77]: def funkcja_a(arg): 
    ...:     """Funkcja z jednym parametrem""" 
    ...:     print(arg) 
    ...:                                                                                                          

In [78]: def funkcja_b(arg, print_data=True): 
    ...:     """Funkcja z parametrem pozycyjnym oraz parametrem nazwanym""" 
    ...:     if print_data: 
    ...:         print(arg) 
    ...:          
    ...:                                                                                                          

In [79]: funkcja_a('siemka')                                                                                      
siemka

In [80]: funkcja_b('siemka')                                                                                      
siemka

Wywoływanie funkcji

Funkcje są obiektami, można je przekazywać i przypisywać do zmiennych. 

In [81]: def dodaj(x, y, return_abs=False): 
    ...:     """Zwraca sumę dwóch argumentów""" 
    ...:     if return_abs: 
    ...:         return abs(x + y) 
    ...:     return x + y 
    ...:                                                                                                          

In [82]: dodaj                                                                                                    
Out[82]: <function __main__.dodaj(x, y, return_abs=False)>

In [83]: zmienna = dodaj                                                                                          

In [84]: zmienna(2, 3)                                                                                            
Out[84]: 5

In [85]: zmienna(2, -3)                                                                                           
Out[85]: -1

In [86]: zmienna(2, -3, return_abs=True)                                                                          
Out[86]: 1

In [87]: zmienna(2, -3, True)                                                                                     
Out[87]: 1

Funkcje mogą obsługiwać nieokreśloną liczbę argumentów pozycyjnych i nazwanych przy użyciu operatorów wyłuskania * oraz **

Moduły

Modułem nazywamy pojedynczy plik z kodem. Duże aplikacje zawierają wiele plików z kodem (wiele modułów). Każdy moduł powinien zawierać kod związany ze sobą (np. drobne funkcje pomocnicze w utils.py, definicje formularzy w forms.py a funkcje walidacyjne w validators.py)

Do wczytywania obiektów zdeklarowanych w innych modułach służy instrukcja import
Można jej używać na kilka sposobów:

  • import modul - obiekty z modulu są dostępne "po kropce" modul.obiekt
  • from modul import obiekt1, obiekt2
  • from modul import * - importuje wszystkie obiekty z modulu (metoda niezalecana)
  • import modul.obiekt as nowa_nazwa - importuje obiekt pod nową nazwą

Python podczas procesu importu wykonuje (jednokrotnie, za pierwszym razem) wszystkie instrukcje zawarte w module (np. instrukcje deklaracji funkcji czy klas). Kolejne wywołanie importu obiektów nie powoduje ich ponownego utworzenia.

Gra saper

Cele i założenia

Gra składa się z pola minowego w rozmiarze 10x10. W losowo wybranych polach znajdują się miny. Odkrycie pola zawierającego minę oznacza jej detonację oraz koniec gry. Pozostałe pola zawierają informację o ilości min w bezpośrednim sąsiedztwie. Celem gry jest poprawne oznaczenie wszystkich pól minowych.

Budowa aplikacji

Kod gry składa się z trzech modułów:

  • game.py - moduł główny do uruchomienia gry
  • mineswp.py - moduł definujący okno gry, ustawienia oraz kontrolki interfejsu graficznego
  • game_lib.py - moduł zawierający funkcje uruchamiane przez akcje użytkownika
Zdefiniowane zdarzenia w grze (np. kliknięcie w pole) uruchamia funkcję, która jest pod to zdarzenie przypisana. Funkcja otrzymuje obiekt zdarzenia, w którym zapisane są informacje o kontrolce graficznej i całym kontekście aplikacji.

Oprogramowanie logiki

Celem ćwiczenia jest uzuepłnienie kodu funkcji z modułu game_lib.py tak, aby spełnić wymagania funkcjonalne gry.
